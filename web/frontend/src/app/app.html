What is this?

The server loads the Mask R-CNN model onto your {{ serverStatus.gpu_name }} and runs a warm-up pass to pre-allocate memory and let cuDNN auto-tune convolution algorithms for the specific input sizes. This ensures the first real inference is just as fast as subsequent ones.

The pipeline uses FP16 mixed precision to leverage tensor cores, batch processing (16 frames at a time), and a pipelined architecture that overlaps disk I/O with GPU compute so the GPU is never idle.
